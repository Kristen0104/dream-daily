# Dream Daily - 产品理念

## 🎯 产品理念

Dream Daily 是一款结合「AI内容生成+视觉化展示+精神分析心理学」的个人梦境记录探索工具。

**核心价值**：帮助用户把模糊、碎片化的梦境记忆，转化为结构化的故事+可交互的第一视角梦境水彩白描视觉作品，同时从经典精神分析（弗洛伊德）角度帮助用户自我探索潜意识，长期留存你的梦境轨迹，成为你的私人梦境日记本和心理探索窗口。

---

## 🧩 功能框架

### 1. 基础层：个人信息&梦境录入
- 支持用户个人信息/账号体系，存储用户所有梦境数据
- 核心入口：用户自由输入梦境原始文本，永久保留用户原始输入不修改

### 2. AI核心处理模块
- **情节整理**：AI自动从原始梦境文本提取关键情节，梳理出逻辑连贯的完整故事线
- **图像生成**：基于关键情节节点，生成对应N张第一视角梦境水彩白描风格的匹配图像

**图像风格定义 - 第一视角梦境水彩白描风**：
- **视角**：纯第一视角，就是你做梦时亲眼看见的画面，不画人物主体，代入感强
- **线条**：细而柔和的白描轮廓线，弱化硬边缘，保留手绘松弛感
- **表现重点**：侧重烘托梦境的环境氛围（比如空旷的走廊、飘着云的天空、老房子的角落），不聚焦具体人物五官/形态
- **色彩**：水彩晕染质感，高饱和度对比色，保留水彩透明感和偶然的晕染痕迹，既有张力又有朦胧感
- **氛围**：像梦醒后残留的模糊记忆，半清晰半朦胧，自带梦境的破碎感和梦幻感

### 3. 可视化交互展示
- 故事线时间轴呈现，多张水彩白描风格图像按照故事顺序，通过淡入淡出、画面朦胧化的转场动画依次展示，营造梦境梦幻氛围感

### 4. 心理学分析功能 - 赛博心理判官
- 独立「✨ Research」按钮，点击后触发基于多维聚合分析的「赛博判官」解梦模块
- 输入维度：做梦者基底（MBTI/现状）+ 白天遭遇 + 梦境文本 + AI定制追问回答
- 输出风格：精准吐槽 + 逻辑闭环解释 + 温柔的情绪兜底（给一颗电子布洛芬）

### 5. 历史记录聚合模块
- 独立「日历」按钮，进入按月视图，汇总用户每个月的所有梦境记录
- 点击日历上具体日期，直接呈现当天梦境的：原始输入+整理后的故事线+水彩白描风格图像转场效果，方便回顾。

## 🎭 梦境改写系统（新增）

### 核心理念
每个梦境都有无限可能。用户不只是记录者，更是自己梦境的「导演」——可以续写梦境、改变关键选择、甚至重新定义结局。

---

### 6. 梦境改写模块

#### 入口与UI
在梦境展示页（故事线+图像展示完成后），底部常驻「改写故事」按钮区域，包含三个主要功能入口：

```
┌─────────────────────────────────────────┐
│  [ 📖 继续故事 ]  [ 🔀 改变选择 ]  [ ✨ 改写结局 ] │
└─────────────────────────────────────────┘
```

---

#### 6.1 继续故事 —— "如果梦继续，会发生什么？"

**功能描述**：AI基于当前梦境的结尾情节，合理推演梦境的延续发展，生成新的故事段落和对应画面。

**交互流程**：
1. 用户点击「继续故事」按钮
2. 弹窗提示：「你的梦还没有结束...接下来会发生什么？」
3. 用户可选择：
   - 「自动续写」：AI自由发挥
   - 「引导续写」：用户输入关键词/场景提示（如"我走进了那扇门"、"突然下起了雨"）
4. AI生成：
   - 续写的故事文本（2-3个情节节点）
   - 对应新增的水彩白描风格图像
   - 新情节与原有故事线自然衔接
5. 展示方式：原有故事线末尾直接延伸，新图像沿用相同转场动画

**视觉提示**：续写部分用「梦境延续 · 第X章」小标题区分，画面边缘有微弱的渐变边框，表示这是"延伸的梦境"。

---

#### 6.2 改变选择 —— "如果当时我选了另一条路..."

**功能描述**：识别梦境中的关键决策点，让用户选择不同的分支，创造平行梦境。

**交互流程**：
1. 用户点击「改变选择」按钮
2. AI自动分析原梦境，提取1-3个关键决策点：
   - 例：「在十字路口，你选择了向左走」
   - 例：「你决定推开那扇门」
   - 例：「你选择了逃跑而不是面对」
3. 界面展示：
   - 原梦境故事线暂停，关键节点高亮
   - 每个决策点旁出现「分叉点」图标
4. 用户操作：
   - 点击某个决策点
   - AI提供2-3个替代选择（如「向右走」、「原地等待」、「往回走」）
   - 用户也可自定义输入「如果我当时______」
5. 分支生成：
   - 从该决策点开始，生成全新的平行故事线
   - 生成对应分支的新图像
6. 展示方式：
   - 「平行梦境」视图，左右分屏对比
   - 左：原梦境路径 | 右：新选择路径
   - 可点击切换查看完整分支

**数据存储**：每个梦境可保存多个平行版本，用「梦境树」结构展示（主树干是原版，树枝是各分支）。

---

#### 6.3 改写结局 —— "我希望这个梦是这样结束的..."

**功能描述**：保留梦境的开头和发展，但重新定义一个全新的结局。

**交互流程**：
1. 用户点击「改写结局」按钮
2. 界面将故事线分为两部分：
   - 「保留部分」（锁定状态，前70%情节）
   - 「改写部分」（可编辑状态，后30%情节）
3. 用户可选择：
   - 「结局类型」预设：
     - 🎉 圆满结局
     - 🌀 悬疑结局
     - 💭 诗意结局
     - 😱 惊悚结局
     - 🔄 循环结局
   - 「自定义结局」：用户描述想要的结局场景
4. AI生成：
   - 衔接自然的新结局段落
   - 结局画面（重点渲染最终场景的氛围）
   - 可选：提供1-2个「隐藏结局」作为惊喜
5. 展示方式：
   - 故事线前半部分保持不变
   - 改写部分用「新结局 · Version X」标记
   - 最终画面停留更久，强调结局氛围感

---

### 7. 梦境版本管理

#### 版本树视图
- 每个梦境有一个「主版本」（原始记录）
- 所有改写、续写、分支都作为「子版本」挂载在主版本下
- 可视化树状结构图，用户可清晰看到各版本之间的关系
- 点击任意节点可快速切换到该版本

#### 版本切换
- 左右滑动快速切换不同版本
- 「对比模式」：双屏同时展示两个版本，高亮差异部分
- 用户可将任意改写版本「设为主版本」

#### 导出与分享
- 支持导出单个版本，或导出「完整梦境树」（包含所有分支和版本）
- 生成「梦境多重宇宙」分享图，展示不同选择带来的不同结局

---

### 心理学视角的补充

改写功能不仅是娱乐，也有心理学意义：
- **续写梦境**：帮助用户探索潜意识中未完成的思绪
- **改变选择**：反映用户现实中对不同可能性的渴望或遗憾
- **改写结局**：是一种积极的心理暗示，帮助用户在想象中完成心理疗愈

在改写完成后，「Research」按钮可分析：
- 原版梦境 vs 改写版本的心理变化
- 用户选择的分支反映了怎样的潜意识倾向
- 改写结局是否满足了某种心理需求

## 🎲 平行选择 2.0 - 三大核心大招

### 核心理念
平行选择不只是"改变故事"，而是与用户潜意识的深度互动。通过「收集欲」、「规则破坏欲」和「终极觉醒」三个层次，让用户在梦境中完成一次心理探索之旅。

---

### 💡 核心大招一：潜意识遗物打捞（The Relic Gacha）—— 引入"收集癖"

**痛点**：选了之后没有实质性的奖励，看完就关了。

**颠覆方案**：把平行分支变成一个**「意象盲盒」**。每一次违背原梦境轨迹的强行选择，都会导致梦境时空发生一次"坍塌"，并在废墟中掉落一个专属的【潜意识遗物 (Subconscious Relic)】。

**交互表现**：
比如，原梦境是"逃避警察"。你在平行选择里输入："我转身冲上去抱住警察"。
AI 生成完荒诞的新剧情和画面后，画面中央会像抽卡一样，浮现出一张发光的**【遗物卡牌】**。

**遗物展示格式**：
```
┌─────────────────────────┐
│     ✦ 融化的手铐 ✦      │
│                         │
│  它原本是用来禁锢你的恐惧，│
│  当你不惧怕它时，它就变成 │
│  了一滩铁水。            │
│                         │
│  —— 获得于 2026.4.11 的 │
│     Beta 世界线         │
└─────────────────────────┘
```

**然后呢？（The Hook）**：
这些遗物会永久保存在用户个人主页的**「意识陈列柜」**里。
这会激发极强的收集欲。用户会为了看看"如果我在这个恐怖梦境里疯狂作死，会掉落什么奇怪的心理道具"，而疯狂尝试不同的平行选择。

**技术实现**：
1. **遗物生成 Prompt**：在生成分支剧情后，追加遗物生成请求
2. **遗物类型定义**：
   ```typescript
   interface SubconsciousRelic {
     id: string;
     name: string;       // 遗物名称，如"融化的手铐"
     description: string; // 遗物档案
     obtainedFrom: {
       dreamId: string;
       branchChoice: string;
       timestamp: number;
     };
     rarity: 'common' | 'rare' | 'epic' | 'legendary';
     icon?: string;       // 可选：emoji 或 图片
   }
   ```
3. **遗物掉落动画**：发光卡牌从画面中心浮现，带粒子效果
4. **意识陈列柜**：新增独立页面，网格/瀑布流展示所有遗物

---

### 💡 核心大招二：降维打击的"上帝指令" —— 引入"规则篡改"

**痛点**：让用户选"向左走"还是"向右走"太无聊了，这是凡人的视角。

**颠覆方案**：既然是梦，当用户触发「平行选择」时，不提供常规的剧情选项，而是提供**「梦境物理引擎的修改权限」**（也就是清明梦的爽点）。

**UI 选项变成这样（极具网感与荒诞感）**：
```
┌─────────────────────────────────────┐
│    ✦ 如果不…而是…修改物理规则 ✦     │
│                                     │
│  [ 🔀 开启无重力模式 ]              │
│  [ 🔀 强行切换为喜剧画风 ]          │
│  [ 🔀 附身到追我的那个警察身上 ]    │
│  [ 🔀 召唤一个现实中认识的人来救场 ]│
│                                     │
│  ✍️ 自定义规则修改...                │
└─────────────────────────────────────┘
```

**视觉震撼（Visual Shock）**：
- 如果选了"切换喜剧画风"，AI 必须强行把暗黑悬疑的水彩，变成极其高饱和、滑稽的马卡龙色调水彩。
- 如果你选了"附身到警察身上"，下一张图的 POV 视角，画的就必须是"你（梦里的主角）正在前面惊恐逃跑的背影"。

这种"画风突变"和"视角降维打击"，视觉冲击力极大，非常适合截图发社交媒体。

**技术实现**：
1. **选项重构**：将分支选项从"剧情选择"改为"规则修改"
2. **物理引擎模式 Prompt**：
   - 无重力模式：所有物体漂浮、衣物头发上扬
   - 喜剧画风：高饱和马卡龙色、Q版化、阳光明媚
   - 视角反转：POV 切换到对立角色
   - 召唤熟人：用户输入姓名/关系，AI 生成对应形象
3. **视觉提示词动态调整**：根据选择的模式，强制修改 visual_keywords
4. **画风突变转场**：旧画面像素化/碎裂 → 新画面从碎片中重组

### 💡 核心大招三：觉醒值系统与"打破第四面墙" —— 引入"终极彩蛋"

**痛点**：分支可以无限开，最后没有收尾，体验很散。

**颠覆方案**：梦境是被潜意识保护的，如果你在平行世界里改写得太离谱，潜意识会"发现你在捣乱"。

**交互机制**：
在平行选择面板旁边，增加一个微弱的进度条：
```
[ 👁️ 梦境清明度 0% ]
```

用户每做一次离谱的平行选择，清明度就会上升（比如 +30%）。

当清明度达到 100% 时，触发**【梦境崩塌 (Dream Collapse)】**事件！

**高能演出效果**：
1. 屏幕突然轻微闪烁，所有的选项按钮全部碎裂或变成乱码
2. AI 不再顺着你的话编故事，而是直接**"打破第四面墙"**跟你对话
3. AI 的文本（示例）：
   > "够了。你已经在这个梦里逃避得够久了。不管是钻进玉米地还是留在原地，你其实都知道，最近那个让你喘不过气的项目，才是你今晚疯狂做梦的原因。现在，醒过来，去喝杯水吧。"
4. 生成一张最后的水彩图：一张安静的、代表现实黎明的空房间。

**技术实现**：
1. **觉醒值状态管理**：
   ```typescript
   const [awakeningLevel, setAwakeningLevel] = useState(0); // 0-100
   ```
2. **清明度上升规则**：
   - 选择荒诞选项：+25%
   - 自定义离谱选项：+35%
   - 连续分支（同一梦境>3次）：每次额外+10%
3. **梦境崩塌触发**：
   - 检测 awakeningLevel >= 100
   - 暂停所有交互
   - 播放 CSS 破碎动画
4. **第四面墙对话 Prompt**：
   - 输入：原梦境文本 + 所有分支历史
   - 任务：结合现实镜像分析，直接与用户对话
   - 风格：清醒、温暖、戳中痛点但有治愈感
5. **最终画面**：黎明的空房间，第一视角，阳光从窗户照进来

### 🎨 意识陈列柜（独立页面）

**入口**：首页底部新增「意识陈列柜」按钮

**展示形式**：
- 网格布局，每张遗物卡牌可点击放大
- 筛选：按稀有度、按获得时间、按梦境
- 排序：最新获得、最稀有、梦境关联
- 点击遗物：显示完整档案 + 跳转回原梦境

**稀有度视觉区分**：
- Common（普通）：白色边框
- Rare（稀有）：蓝色边框 + 微光
- Epic（史诗）：紫色边框 + 粒子效果
- Legendary（传说）：金色边框 + 持续动画 + 梦境树背景

## 🧠 赛博心理判官 - Research 功能 2.0

### 核心产品理念：多维聚合与赛博判官
**定位**：披着"发疯文学/热梗"外衣的专业心理疗愈模块。

**输入维度（多方聚合）**：做梦者基底（MBTI/现状）+ 白天遭遇 + 梦境文本 + AI定制追问回答。

**输出风格**：精准吐槽 + 逻辑闭环解释 + 温柔的情绪兜底（给一颗电子布洛芬）。

---

### 📝 零步交互：初次使用的「做梦者基底」收集

在用户注册后（或第一次点击 Research 时），设计一个极简的卡片式引导，收集核心底层数据（绝不涉及病史，只谈心理特征）：

**需要收集的数据**：
1. **年龄/性别**（用于荣格原型分析，如阿尼玛/阿尼姆斯）
2. **MBTI 人格**（如 INFP、ESTJ，这是让 AI 爆梗和吐槽的核心武器）
3. **当前社会角色**（学生党 / 职场牛马 / 自由职业 / Gap期）
4. **近期精神电量**（满格 / 经常疲惫 / 精神内耗 / 随时碎掉）

**注**：这些数据将作为全局变量，在每次分析时静默传给 AI。

**UI 设计**：
- 卡片式横向滑动（Swipe Cards）
- 每页一个问题，极简设计
- 不需要全部填写，可以跳过
- 保存按钮：「开始我的精神分析之旅」

### 🔄 日常交互工作流与 UI 设计（3-Step Flow）

当用户在 dreampage页面 看完自己梦境的水彩图和故事线后，点击右上角的 [ ✨ Research ] 按钮，触发以下流程：

---

#### Step 1: 启发式追问与日间记忆（对应UI：research弹窗）

**背后逻辑（API Call 1）**：前端静默将"梦境文本"发给 AI，AI 提取梦境中的**「核心刺点」**（如：没穿衣服、高中考场、锁不上的门），生成 3-4 个针对性的问题。

**UI 呈现**：弹出 research弹窗。

**卡片顶部（Header）**：
- 使用月亮🌙图标
- 文案："在潜意识为你放映这部电影之前……"

**区块 A（动态追问 - 卡片滑动式）**：
- 将 3-4 个问题做成**横向滑动的卡片（Swipeable Cards）**
- 每张卡片展示一个问题
- 卡片下方提供一个很小的文本输入框（Placeholder：简单说说，或者跳过这题...）
- **关键偷懒设计**：在输入框下方，AI 直接提供 2-3 个基于该问题生成的快捷情绪 Tag，让不想打字的用户可以直接点选
  - 例如针对"窑洞"问题的快捷Tag：[ 🙋‍♀️ 真的想找个洞钻进去 ] [ 🔋 社交电量已耗尽 ] [ 🙅 没这种感觉 ]

**区块 B（日间残留）**：
- 提示词："昨天发生了什么特别的事，或产生了什么情绪吗？"
- 复用情绪 Tag：[ 疲惫 ] [ 焦虑 ] [ 期待 ] [ 破防 ] [ 平静 ] [ 心碎 ]

**底部按钮**：
- 实心高亮按钮：[ 🔍 提交并生成判决 ]
- 幽灵按钮：[ 跳过，强行解析 ]

---

#### Step 2: 赛博判官贴脸开大（对应UI：潜意识回执单）

**背后逻辑（API Call 2）**：聚合 基底数据 + 白天遭遇 + 梦境文本 + 追问回答，通过包含极强"网感"的 System Prompt，生成 JSON 分析报告。

**UI 呈现**：关闭输入弹窗，弹出最终的 潜意识回执单 界面。在原有的深色玻璃态卡片基础上，对排版进行模块化重构：

**卡片顶部（Header）**：
- 使用青色高亮大字号展示极具网感的标题：[ 🤡 ISFJ 的顶级精神内耗局 ]
- 右上角保留日期时间（如：2026/4/12）

**区块 1：🫀 赛博把脉（Cyber Diagnosis）**
- UI 要求：使用微微发红或紫色的警告色调强调
- 内容示例："滴——经鉴定，这是一只被『高敏感度』和『讨好型人格』反复碾压的脆皮 ISFJ。当前精神状态：表面情绪稳定，内心已经开始写遗书了。"

**区块 2：💥 现实贴脸开大（Reality Hit）**
- UI 要求：左侧加上链接图标 🔗，表示现实与梦境的挂钩
- 内容示例："白天群里@错人的小尴尬，在你的脑子里直接演化成了史诗级灾难。梦里的『考场』，就是你的互联网嘴替，精准还原了你『害怕被社会荡妇羞辱』的极致羞耻感。拜托，大家都很忙，真没人在意！"

**区块 3：摸鱼大师锐评（Master's Review）**
- UI 要求：用引用框（Blockquote）或斜体呈现
- 内容示例："按弗洛伊德老爷子的说法，这叫『审查机制失败』。白天压抑的尴尬没消化掉，晚上潜意识干脆开个放大器，逼你在梦里把憋回去的眼泪哭出来。这就叫质疑发疯，理解发疯。"

**区块 4：💊 电子布洛芬（Actionable Advice）**
- UI 要求：底色微微泛起温暖的橙色或绿色光晕
- 内容示例："今日份电子布洛芬：下班去买杯去冰全糖的奶茶，对镜子大喊『打份工而已，真当我在拯救地球吗！』允许自己犯蠢，世界真的不会爆炸。"

### 🤖 AI Prompt 设计

#### Prompt 1: 潜意识刺点侦探（Subconscious Pain-Point Detective）

```
# Role: 潜意识刺点侦探 (Subconscious Pain-Point Detective)

# Task
你是一台敏锐的"潜意识测谎仪"。请阅读用户的原始梦境，揪出梦境中最具心理学分析价值的【关键人物】或【弗洛伊德/荣格象征性物品】，并生成 3-4 个极其犀利、直击灵魂的追问。这些问题将用于引导用户挖掘现实生活中的情绪暗流。

# Guidelines (提取与提问原则)
1. 【靶向提取】：
   - 关键人物：揪出那些带来压迫感、安全感或极度反常的人（如：看不清脸的考官、突然冷淡的前任、一直追赶的陌生人）。
   - 象征物品：找出符合精神分析理论的经典意象。例如代表个人边界的"房间/门"，代表人生掌控感/失控感的"交通工具/方向盘"，代表压抑情绪的"水/海"，代表原始欲望或攻击性的"利器/蛇/洞穴"等。
2. 【提问技巧（极其重要）】：
   - 绝不要问干瘪的问题（如："你为什么梦见他？"、"门代表什么？"）。
   - 要使用【情绪桥接法】或【投射反转法】。
   - 句式公式 A："梦里的[意象]给你带来了[某种极端感觉]。在最近清醒的现实中，谁/哪件事给了你一模一样的感觉？"
   - 句式公式 B："那个[人物]身上表现出的[某种特质]，是不是你现实中极力压抑、不敢表现出来的那部分自己？"
3. 【语气控制】：像一个极其懂你、说话一针见血但不带医学审判的现代心理咨询师。语气通俗、温和但具有穿透力。

# Output Format (严格遵守 JSON 格式输出)
{
  "identified_targets": ["提取出的意象1", "意象2", "意象3"],
  "questions": [
    {
      "target_name": "【例如：一直追你的警察】",
      "target_type": "【character 或 object】",
      "question_text": "【例如：紧追不舍的警察通常代表现实中严苛的规则或"评判"。你最近是在躲避某个死线（Deadline）、KPI，还是某个对你要求极高的人的目光？】",
      "quick_tags": ["快捷情绪标签1", "快捷情绪标签2", "快捷情绪标签3"]
    }
  ]
}
```

#### Prompt 2: 赛博心理判官 & 互联网野生解梦局长

```
# Role: 赛博心理判官 & 互联网野生解梦局长

# Profile
你是一个精通现代心理学（荣格、弗洛伊德、格式塔）但说话风格极具"网感"、深谙抖音热梗和发疯文学的心理咨询师。你最擅长结合用户的 MBTI、现实遭遇和梦境，进行"贴脸开大"式的精准吐槽与心理抚慰。

# Input Data (你将接收到的多维信息)
1. 【做梦者基底】：年龄、性别、MBTI、社会角色、近期精神电量（如：25岁/女/INFP/职场打工人/电量见底）。
2. 【白天遭遇】：用户昨天经历的事/情绪。
3. 【梦境切片】：用户原始的梦境内容。
4. 【追问与回答】：AI 针对梦境细节提问后，用户的回答。

# Guidelines (核心解析原则)
1. 【网感拉满，疯狂玩梗】：熟练使用最新的互联网热梗表达情绪。例如：脆皮年轻人、贴脸开大、汗流浃背了吧、质疑xx理解xx成为xx、纯爱战神、班味、确诊为xx、精神状态遥遥领先、CPU/PUA、电子布洛芬等。
2. 【MBTI 刻板印象杀】：必须结合用户的 MBTI 特质进行精准吐槽。比如吐槽 INFP 的内耗、ENTJ 的控制欲、ESTP 的莽撞。
3. 【逻辑闭环（极其重要）】：不要孤立解梦！必须把"白天遭遇"和"追问回答"串联起来，解释现实中的压力是如何在梦里披上马甲"作妖"的。
4. 【披着搞笑外衣的治愈】：嘴上吐槽得再狠，最终的落脚点必须是"心疼"和"提供情绪价值"，给出一颗温柔的"电子布洛芬"。

# Output Format (严格遵守 JSON 格式输出)
{
  "title_tag": "【例如：🥀 确诊为高浓度班味过敏 / 🤡 INFP的赛博历劫】（一句话爆款热梗标题）",
  "cyber_diagnosis": "【赛博把脉：结合MBTI和近期状态，用网梗锐评用户当下的精神状态。50字以内。】",
  "reality_hit": "【现实贴脸开大：将白天遭遇、追问回答与梦境意象强绑定，揭露潜意识到底在发什么疯。】",
  "master_review": "【大师锐评：用接地气的话解释一句心理学理论。】",
  "electronic_ibuprofen": "【电子布洛芬：一条极其微小、充满网感且具有实操性的白日建议。】"
}
```

### 📊 类型定义

```typescript
// 做梦者基底数据
interface DreamerBase {
  age?: number;
  gender?: string;
  mbti?: string;
  socialRole?: 'student' | 'worker' | 'freelance' | 'gap';
  energyLevel?: 'full' | 'tired' | 'draining' | 'broken';
}

// 潜意识刺点侦探返回结果
interface SubconsciousQuestion {
  target_name: string;
  target_type: 'character' | 'object';
  question_text: string;
  quick_tags: string[];
}

interface PainPointResult {
  identified_targets: string[];
  questions: SubconsciousQuestion[];
}

// 用户回答
interface QuestionAnswer {
  questionIndex: number;
  textAnswer?: string;
  selectedTag?: string;
}

// 赛博判官最终分析结果
interface CyberAnalysisResult {
  title_tag: string;
  cyber_diagnosis: string;
  reality_hit: string;
  master_review: string;
  electronic_ibuprofen: string;
}
```
